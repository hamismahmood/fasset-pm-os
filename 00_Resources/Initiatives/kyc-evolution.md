## Context
Migrating from Jumio to Sumsub. PoA collection moved upfront. Multiple alternatives evaluated.

## Current state
- Jumio → Sumsub migration planned (3-4 month phased rollout, Altof owns).
- Both systems will run during migration to avoid forcing users to redo KYC.
- PoA collected upfront after advanced KYC if user wants IBAN or card.
- KYC Reliance Model: Keel using Jumio data once Due-side changes complete.

## Issues
- PoA success rate only 20-30% (missing identity document pictures, laminated docs rejected, >90 day docs rejected).
- KYC status miscommunication (verified users seeing "in progress" until cache cleared).
